# PO Prompt: Codebase-Enhanced Task Analysis

> **Use Case**: Analyze new features for existing projects using task definition AND project codebase

## When to Use

- Existing projects with codebase
- New feature implementation
- Sprint planning with effort estimation

## Prompt

Analyze the task and provide:

### 1. Business Gaps
- Missing requirements needing PO clarification
- Edge cases and complex scenarios

### 2. Impact Analysis 
- Impacted components (DB, backend, frontend, APIs)
- Regression testing areas

### 3. Development Tasks
| Task | Component | Effort | Dependencies |
|------|-----------|--------|--------------|
| ... | ... | Xh | ... |

**Total Estimate:** X hours (Dev + QA)

### 4. Risks & Acceptance Criteria
- Key risks with mitigations
- Definition of done checklist


## Expected Output

1. **Business Gaps**: Missing requirements and edge cases
2. **Impact Analysis**: Affected components and regression areas
3. **Development Tasks**: Detailed task breakdown with effort estimates
4. **Risks & Acceptance Criteria**: Risk mitigation and DoD checklist
