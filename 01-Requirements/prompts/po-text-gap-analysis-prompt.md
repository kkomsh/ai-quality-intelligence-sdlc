# PO Prompt: Text-Only Gap Analysis

> **Use Case**: Analyze requirements when there is no existing codebase (new project or early phase)

## When to Use

- Project initial phase with no code yet
- Standalone feature analysis
- Early requirements review

## Prompt

Analyze the task and provide:

1. Business Gaps
   - Missing requirements needing clarification

2. Edge Cases
   - Complex or rare scenarios to handle

3. Fault Management
   - Failure points and error handling strategies


## Expected Output

- List of business gaps requiring PO clarification
- Identified edge cases and complex scenarios
- Failure points with recommended error handling strategies
